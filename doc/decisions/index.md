## ADR-001 —— 技术栈选型

### 背景

屋檐项目需要从零搭建技术框架，需要选择合适的前后端技术栈和数据库。

### 决策

| 层级 | 选择 | 备选方案 |
|---|---|---|
| 前端 | React 18 + TypeScript + Vite | Vue 3 + Vite |
| 后端 | Python + FastAPI | Flask / Django |
| 数据库 | SQLite | PostgreSQL / MySQL |

### 原因

- **React + TypeScript**：类型安全、生态成熟、组件化开发效率高
- **FastAPI**：异步性能、自动API文档、Pydantic数据验证
- **SQLite**：MVP阶段零配置、足够使用、后续可迁移

### 影响

- 前端使用函数式组件 + Hooks 开发模式
- 后端使用异步API设计，便于后续扩展
- 数据库设计需考虑后续迁移兼容性

## ADR-002 —— 项目文档规范

### 背景

项目需要建立统一的文档规范，便于产品、开发和后续维护。

### 决策

采用 doc 目录结构规范：

```
doc/
├── product/      # 产品层（愿景、路线图、变更日志）
├── requirements/ # 需求层（需求看板、需求文档、Bug文档）
├── design/       # 设计层（架构、数据模型、API、页面设计）
└── decisions/    # 决策记录（ADR）
```

### 原因

- 清晰的层级划分，便于不同角色查找信息
- 模板化文档，降低维护成本
- 支持 Mermaid 图表，可视化表达

### 影响

- 所有需求必须通过 REQ-xxx 文档记录
- 所有技术决策必须通过 ADR 记录
- 文档与代码同步更新
